# Project Overview
![dashboard main](/img/main.png)
The MP/MLA Constituency Management System is a digital governance platform designed to streamline the complex operational responsibilities of elected representatives, including managing citizen requests, tracking projects, monitoring fund utilization, keeping a track of all active departments, and generating reports.
Traditionally, these tasks were handled through fragmented methods like physical registers and spreadsheets, leading to inefficiencies such as delayed grievance resolution, poor budget visibility, and limited accountability. This system addresses these challenges through a centralized, cloud-ready web platform that integrates all constituency operations.
It is built on three key principles:
- Centralization — consolidates all data into a single platform, eliminating duplication and silos
- Role-Based Access (RBAC) — ensures secure, role-specific access for MLA/MPs, staff, and administrators
- Data-to-Decision Flow — enables real-time data capture, processing, and visualization for informed decision-making

The system includes 18 interconnected modules such as Dashboard, Projects, Public Requests, Funds Utilization, Public Facilities, Demographics, Departments, Wards, Community Groups, Local Representatives, Birthdays, Meetings and Events, Reports, all linked through ward-based mapping. Advanced Modules like Audit Logs, Permissions, Recycle Bin, Settings, User Management accessible to MP/MLA or System Admin only.

A key feature is the Public Registration System, allowing institutions to self-register with KYC verification, subject to MLA approval, ensuring data authenticity and trust.

Additional capabilities include SMS and WhatsApp communication, XLSX report exports, recycle bin to recover mistakenly deleted records, bulk data import, and detailed audit trails.

Security is maintained through role-based permissions, session controls, and limits on failed login attempts.
