### Purpose
This module allows the System Administrator to control what actions each role in the system can perform. It defines access rights for different modules and features by enabling or disabling permissions such as Create, Read, Update, Delete, and Export. This ensures that users only have access to the functions required for their role, improving security and system control.
### How to Use
-	Log in as System Administrator or MP/MLA.
-	From the Administration section in the left navigation panel, click Permissions.
-	The system will display available roles such as System Admin, MLA / MP, and Office Staff.
-	Select the role whose permissions you want to configure.
-	The system will display a list of modules (such as Audit Logs, Backups, Branding, etc.) with available permission options.
-	For each module, enable or disable permissions such as:
    -	Create – Allows the user to add new records.
    -	Read – Allows the user to view data.
    -	Update – Allows the user to edit existing records.
    -	Delete – Allows the user to remove records.
    -	Export – Allows the user to export system data.
-	Toggle the permission switches on or off based on the access level required for the selected role.
-	Once permissions are configured, the system automatically applies the updated access rules to users assigned to that role.
